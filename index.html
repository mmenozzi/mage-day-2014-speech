What Composer is


Composer is a tool for dependency management in PHP. It allows you to declare the dependent libraries your project needs and it will install them in your project for you.

Composer è un tool open source, sviluppato in php, per la gestione delle dipendenze in progetti php. Le sue potenzialità lo hanno fatto diventare rapidamente lo standard de facto in questo ambito. Si pensi semplicemente al fatto che alcune librerie storiche, come phpunit, stanno abbandonando pear per passare a composer come sistema di distribuzione. Considerando che pear è stato per tanti anni il sistema ufficiale in php per distribuire librerie si capisce quanto questo strumento sia rivoluzionario.

What Composer does…


/composer.json 


{
    "require": {
        "php": ">=5.3.0",
        "psr/log": "~1.0",
        "acme/foo": "@dev",
        "monolog/monolog": "dev-master#2eb0c0978d290a1c45346a1955188929cb4e5db7",
    },
    "require-dev": {
        "phpunit/phpunit": "~3.7.0",
        "graylog2/gelf-php": "~1.0",
        "doctrine/couchdb": "~1.0@dev",
    },
    "repositories": [
        {
            "type": "vcs",
            "url": "git@github.com:AcmeCompany/FooLibrary.git"
        },
    ]
}

$ composer install

Composer permette di definire in un file json (composer.json) l'elenco dei pacchetti da cui il nostro progetto dipende; per ogni pacchetto può essere specificato un vincolo di versione o di stabilità. Lanciando poi un singolo comando, Composer utilizza questi metadati per scaricare ed installare per noi queste dipendenze all'interno del nostro progetto. L'aspetto interessante è proprio in queste ultime parole "nel nostro progetto", infatti su un secondo progetto avremo un altro Composer.json con diverse dipendenze che saranno utilizzate per quel progetto. Altra cosa importante da tenere presente è che Composer, ci permette di installare non solo librerie pubbliche ma anche dipendenze non open-source che si trovano ad esempio su repository privati.

… and how



  1. Available packages indexing
  2. Dependency tree and packages list calculation
  3. Packages list locking (composer.lock file)
  4. Packages download (in vendor folder)
  5. Custom installers
  6. Autoload dump
Quando lanciamo l'installazione delle dipendenze, Composer scansiona il repository packagist.org (più eventuali repository aggiuntivi definiti nell'apposita sezione del Composer.json) e indicizza i pacchetti disponibili, le loro versioni e le loro dipendenze. Dopodiché risolve l'albero delle dipendenze in maniera automatica definendo così un elenco di pacchetti da installare ognuno ad una specifica versione. Questo elenco viene "lockato" nel file composer.lock per fare in modo che in futuro, per quel progetto, vengano sempre installate quelle dipendenze a quelle esatte versioni. I pacchetti vengono poi scaricati utilizzando le i metadati definiti nei composer.json di ogni pacchetto e i relativi file vengono salvati in una cartella all'interno del nostro progetto (che solitamente si chiama "vendor"). Come penultimo passo, vengono eseguiti eventuali custom installers che permettono di eseguire operazioni custom di installazione. Vedremo tra poco che è proprio un particolare custom installer che ci consente di usare Composer su Magento. Infine, Composer, genera per noi, in automatico, un unico autoloader che, se utilizzato, ci fa l'autoload di tutte le classi presenti nei pacchetti appena scaricati. Tutto questo in poche decine di secondi!

Benefits



  • Time save
  • Code reuse
  • Code sharing
  • Easy upgrades
I vantaggi di un tool di questo tipo sono molteplici: Risparmio di tempo: innanzi tutto se dovessimo fare il lavoro appena descritto manualmente tutte le volte che ne abbiamo bisogno capiamo bene che il tempo che impiegheremmo sarebbe molto più alto.
Ri-usabilità del codice: se nel nostro progetto ci serve un sistema di logging affidabile è inutile che ci scriviamo da zero l’ennesimo motore di logging! Ce ne sono già diversi e fatti molto bene quindi usiamoli. Con Composer è molto facile perché basta scegliere la libreria di logging che vogliamo utilizzare e aggiungerla alle dipendenze del nostro progetto.
Condivisione del codice: se invece, per un particolare progetto, scrivessimo una libreria particolarmente utile e ri-usabile, tramite Composer possiamo facilmente fare in modo che altri la utilizzino. Chissà magari poi la community ce la può pure migliorare.
Aggiornamenti facili: se una delle librerie da cui il nostro progetto dipende ha rilasciato una nuova versione che ci farebbe dannatamente comodo possiamo con un singolo comando (composer update) aggiornare la libreria anche nel nostro progetto.
